Today Scripts - Make your Mac notice center more practical

2017-08-07
Mac, Monitor, Notice center

Today Scripts is a small tool that runs bash scripts in the notification center and outputs the results. You can use it to view system information, performance data, resource usage, read RSS, management server. As long as you can use the terminal command to output the results of things, you can use it to achieve. But also free of charge.

Presumably many people have used iStat menus, because the need to show on the menu bar, will occupy the already very crowded menu bar space, if you do not need real-time monitoring, you can choose to use this tool in the notification center to do the same work. Moreover, the notification center gadget will only take up system resources when it is displayed.

Some students may know that this software was originally run on OS X 10.10, and the minority had published a recommendation at the end of 2014. For unknown reasons, the original author SamRothCA did not provide the adaptation of the new system, resulting in OS X 10.11 update after the software scrapped, after lsd released for the 10.11 adapter version. Into the macOS era, Apple modified the notice center design style, resulting in no adaptation of the Widget display is not coordinated. So I have to modify the code on the new system has been adapted, and now you can already in the macOS 10.12 on the normal use of the software.

Basic use

To add a script to Today Scripts, just click on the “i” in the upper right corner and add the script.

Above you can enter the title of the script and call the program (the default will choose the system currently used shell). No special needs, then in the middle of the box, enter the bash script and click “add script” can be. Note that you need to press the Tab key to move the cursor between the text boxes. (Shift Tab can move in the opposite direction)

  • Run automatically: Does not run the script automatically if unchecked. You need to click the mouse on the script title to output the result. You can use this option to do some click on the implementation of the shortcut tool.
  • Show status indicator: The normally added script will display a small light indicating the running status on the left side of the title, and uncheck it will not be displayed.

With iStats temperature monitoring

You must have used some temperature monitoring tools, here is a combination of command line tools iStats implementation.

First install iStats. Use gem install iStats to install directly. After installation, run the istats on the terminal to display the results.

Then you only need to add a content in Today Scripts / usr / local / bin / istats script on the line. The CPU temperature, fan status, and battery status are displayed by default. Next I will introduce the way to customize the display data.

In the terminal input istats scan to retrieve the existing temperature sensor, the program will list a bunch of know why and do not know why the temperature information.

You can choose from what you need to join, with istats enable xxx can. For example, I would like to join the last AirPort temperature, we must enter istats enable TW0P. Then you can see the newly added sensor under the Extra Stats category.

If you only want to see some of the data in the notification center, you can use in Today Scripts such as /usr/local/bin/istats cpu, /usr/local/ bin/istats battery, etc. to limit the content, pay attention to the front must use iStats absolute Path, more commands can be entered in the terminal istats help to view.

Run a non-Bash script

In addition to running the default bash script, we can also run scripts in some other languages. Only need to add the script when the call to the corresponding executable file can be changed, pay attention to the absolute directory.

  • Python: /usr/bin/python
  • Ruby: /usr/bin/ruby
  • Apple Script: /usr/bin/osascript (Tip: In the Apple Script you can use the log statement to output text to the console)
